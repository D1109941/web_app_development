# 產品需求文件 (PRD)

## 專案名稱
食譜收藏夾

## 專案描述
這是一個專為愛料理的讀者及主廚設計的食譜收藏與管理網站，協助使用者輕鬆尋找理想食譜、記錄收藏並追蹤食材準備情況，解決料理前備料手忙腳亂與找不到好食譜的痛點。

## 1. 專案概述
- **背景與動機**：現代人多半在家下廚，但常常面臨「不知道煮什麼」、「忘了某道菜怎麼煮」或是「買錯/漏買食材」的問題。本系統旨在幫助使用者有效率地搜尋、收藏食譜並管理備料。
- **目標用戶**：
  - **一般使用者**：想找食譜、收藏食譜、記錄食材準備清單的料理愛好者。
  - **管理員**：負責管理平台上的系統資訊、更新或維護食譜資料庫。
- **核心價值主張**：提供直覺流暢的食譜搜尋與收藏體驗，加上便利的食材準備清單功能，讓下廚變得更輕鬆有趣。

## 2. 功能需求
以下列出本系統的主要功能與對應的使用者故事：

### 功能一：搜尋食譜
- 作為 **一般使用者**，我希望 **能用關鍵字或分類搜尋食譜**，以便 **快速找到我今天想煮的菜色**。

### 功能二：瀏覽食譜
- 作為 **一般使用者**，我希望 **能流暢地瀏覽食譜的圖文與步驟**，以便 **確認這道菜的烹調細節是否符合我的需求**。

### 功能三：儲存食譜
- 作為 **一般使用者**，我希望 **能將中意的食譜儲存（收藏）起來，並可在個人頁面查看**，以便 **建立我的專屬料理秘笈，未來快速複習**。

### 功能四：食材準備清單
- 作為 **一般使用者**，我希望 **能把所需食材加入準備清單**，以便 **去超市採買時當作 Checklist，避免漏買任何一項**。

### 功能五：計時功能
- 作為 **一般使用者**，我希望 **在烹調時能使用系統內建的計時功能**，以便 **精準掌握火候與每道手續所需時間，不至於手忙腳亂**。

### 功能六：會員註冊與登入功能
- 作為 **一般使用者 / 管理員**，我希望 **能註冊並安全地登入帳號**，以便 **確保我儲存的食譜與個人設定被妥善保護**。

### 功能七：食譜庫管理（管理員限定）
- 作為 **管理員**，我希望 **能夠新增、編輯與刪除平台上的公開食譜**，以便 **維持平台內容的豐富度與正確性**。

## 3. 非功能需求
- **技術限制**：
  - 後端架構：Python + Flask
  - 模板引擎：Jinja2
  - 資料庫：SQLite
- **效能與安全考量**：
  - 密碼需透過安全機制（如 Hash 雜湊）處理儲存。
  - 登入狀態應使用 Session 並適當檢查使用者權限。
  - 防範常見的 SQL Injection 攻擊以保障資料正確性。

## 4. MVP 範圍
- **Must Have (必須有)**：
  - 基本的會員註冊與登入
  - 瀏覽食譜與關鍵字搜尋
  - 儲存食譜 (加入收藏)
  - 簡易版的食材準備清單追蹤
  - 管理員能新增/刪除食譜
- **Should Have (應該有)**：
  - 在食譜頁面內建烹飪計時功能
  - 食譜的分類標籤 (例如：中式、西式、甜點)
  - 個人儲存清單的整理與篩選
- **Nice to Have (最好有)**：
  - 允許一般使用者自行上傳並分享食譜
  - 對食譜進行留言與評分
